# 项目边界

## 在范围内

- Ego 模型规范（基座选型、LoRA 结构、edition 体系）
- Ego Pod — 全自动化训练管线
- 数据采集器 — 从指定数据源读取数据（定义输入格式，不定义数据本身的格式）
- 语料构建器 — 将原始数据转化为训练语料
- 训练引擎 — 基于基座训练 LoRA 适配器
- 版本管理 — 多 edition 输出、旧版本覆盖
- 独立工具模式 — 脱离 iFay 可独立使用

## 不在范围内

- Fayger（虚拟运行时环境）
- Faying（加载协议）
- FayID（身份体系）
- Sense/Action/Thought/Skill/Credentials 的数据格式定义（由姐妹项目定义）
- 协议层（Faying Protocol、Telepathy Protocol 等）

Ego 只定义"我需要什么格式的输入"以及"我输出什么"。数据从哪来、怎么送到终端，是姐妹项目的事。

> 类比：Ego Pod 就像一个"厨师"——你告诉它食材在冰箱的哪一层（数据源地址），它负责做菜（训练）。但冰箱怎么设计、食材怎么采购，那是别人的事。
