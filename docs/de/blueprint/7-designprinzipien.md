# Designprinzipien

1. **Sofort einsatzbereit** — Normale Nutzer können nach `pip install` mit drei Befehlen loslegen. Kein Machine-Learning-Wissen oder GPU-Treiber-Konfiguration erforderlich.
2. **Vollautomatisch** — Von der Datensammlung bis zur Modellausgabe ist kein manueller Eingriff nötig. Wie eine Waschmaschine: Wäsche rein, Start drücken, saubere Wäsche rausnehmen.
3. **Multi-Versions-Anpassung** — Ein Trainingslauf erzeugt mehrere Versionen für unterschiedliche Rechenkapazitäten. Das gleiche „Du" kann gleichzeitig auf einer Smartwatch, einem Handy und einem Computer leben.
4. **Wiederholbare Ausführung** — Nach Datenaktualisierungen neu trainieren, und Ego entwickelt sich entsprechend weiter. Wenn du wächst, wächst Ego mit.
5. **Austauschbare Basis** — Bei Erscheinen neuer Basismodelle wechseln und LoRA neu trainieren. Wie ein Prozessor-Upgrade beim Handy — die Leistung verbessert sich, aber die „Persönlichkeit" bleibt gleich.
6. **Plugin-erweiterbar** — Datenquelladapter, Basismodelle und Editionen sind alle erweiterbar. Die Community kann neue Datenquellen-Plugins beitragen (z. B. Notion-Adapter, WhatsApp-Chatverlauf-Importer).
7. **Privacy First** — Alles Training kann lokal abgeschlossen werden, Daten verlassen niemals das Gerät. Dein Tagebuch, Chatverlauf und deine Arbeitsnotizen bleiben für immer auf deinem eigenen Computer.
