# 專案邊界

## 在範圍內

- Ego 模型規範（基座選型、LoRA 結構、edition 體系）
- Ego Pod — 全自動化訓練管線
- 資料採集器 — 從指定資料來源讀取資料（定義輸入格式，不定義資料本身的格式）
- 語料建構器 — 將原始資料轉化為訓練語料
- 訓練引擎 — 基於基座訓練 LoRA 適配器
- 版本管理 — 多 edition 輸出、舊版本覆蓋
- 獨立工具模式 — 脫離 iFay 可獨立使用

## 不在範圍內

- Fayger（虛擬執行環境）
- Faying（載入協定）
- FayID（身份體系）
- Sense/Action/Thought/Skill/Credentials 的資料格式定義（由姊妹專案定義）
- 協定層（Faying Protocol、Telepathy Protocol 等）

Ego 只定義「我需要什麼格式的輸入」以及「我輸出什麼」。資料從哪來、怎麼送到終端，是姊妹專案的事。

> 類比：Ego Pod 就像一個「廚師」——你告訴它食材在冰箱的哪一層（資料來源位址），它負責做菜（訓練）。但冰箱怎麼設計、食材怎麼採購，那是別人的事。
