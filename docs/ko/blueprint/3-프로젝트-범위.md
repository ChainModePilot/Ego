# 프로젝트 범위

## 범위 내

- Ego 모델 사양 (베이스 모델 선정, LoRA 구조, edition 체계)
- Ego Pod — 완전 자동화 훈련 파이프라인
- 데이터 수집기 — 지정된 데이터 소스에서 데이터 읽기 (입력 형식 정의, 데이터 자체의 형식은 정의하지 않음)
- 코퍼스 빌더 — 원시 데이터를 훈련 코퍼스로 변환
- 훈련 엔진 — 베이스 모델 기반 LoRA 어댑터 훈련
- 버전 관리 — 다중 edition 출력, 이전 버전 덮어쓰기
- 독립 도구 모드 — iFay 없이 독립 사용 가능

## 범위 외

- Fayger (가상 런타임 환경)
- Faying (로드 프로토콜)
- FayID (아이덴티티 시스템)
- Sense/Action/Thought/Skill/Credentials의 데이터 형식 정의 (자매 프로젝트에서 정의)
- 프로토콜 레이어 (Faying Protocol, Telepathy Protocol 등)

Ego는 "어떤 형식의 입력이 필요한지"와 "무엇을 출력하는지"만 정의합니다. 데이터가 어디서 오는지, 어떻게 단말에 전달하는지는 자매 프로젝트의 영역입니다.

> 비유: Ego Pod는 "셰프"와 같습니다——식재료가 냉장고 어느 칸에 있는지(데이터 소스 주소)를 알려주면, 요리(훈련)를 담당합니다. 하지만 냉장고 설계나 식재료 구매는 다른 사람의 일입니다.
